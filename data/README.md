# LabLine community dataset

An **open, company-level** directory of clinical laboratories and diagnostics/biotech firms,
built from public sources. It exists so scientists job-hunting in this field can share what
they learn about *organizations* — not to compile or pass around data about people.

## What's in it

`community-dataset.sample.json` is a small hand-built **seed** proving the schema. Real
entries are generated by the LabLine app on a contributor's own machine (Research & Prep →
tick **"Sanitized for sharing"** → **Download → Community dataset (open)**), then merged here.

Each entry:

| Field | Source |
| --- | --- |
| `name`, `city`, `state`, `zip`, `certificate`, `facility_phone`, `external_id`, `listed_in` | Public-domain U.S. federal files (CMS CLIA, NPPES, openFDA) |
| `profile` | A short, **cited** company summary drawn from public web sources |
| `profile_sources`, `researched` | The source URLs used, and the date |

## Hard rules (what this dataset is NOT)

- **No personal data.** No names, titles, roles, emails, or direct lines of any individual.
  The generator is prompted to refuse naming people, and the export schema has no field for it.
- **No private notes or outreach pipelines.** Those never leave a contributor's browser.
- **Company-level facts only**, each grounded in a cited public source. Unverified — confirm
  before relying on any entry.

This mirrors the project's stance in [`../docs/DATA-SOURCES.md`](../docs/DATA-SOURCES.md):
public federal registries and public company facts are fair to share; scraped personal
contact data is not, and never will be.

## Contributing

1. Run LabLine locally, load your state, filter to the labs you researched.
2. Research them with **"Sanitized for sharing"** ticked.
3. Download the **Community dataset (open)** JSON and open a pull request adding your entries
   to `community-dataset.json` (dedupe by `name` + `city` + `state`).
4. Entries that contain any personal data will be rejected.
