Browsers can't call ollama.com directly (CORS). Run tools/ollama-search-proxy.py and point here: http://127.0.0.1:8791/search (use 127.0.0.1, not localhost). The proxy holds your key — nothing sensitive in the browser.
Leave blank when using the proxy (safer — the key stays in the proxy, not this browser). Stored only in this browser if set.
One-time setup: so the browser may talk to your local model, start Ollama allowing this origin — e.g. set OLLAMA_ORIGINS=* before launching ollama serve. This works when you run LabLine locally; the public site can't reach a local model.
